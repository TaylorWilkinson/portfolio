brief

Design, code and present a responsive website about retrofitting your home, based on data provided by City Studio Vancouver. This was a group course project for Information Design course (IAT 235). 2015.

coding

All team members contributed to writing equal portions of the website's code. I coded environment page and it's two subsections. Additionally, I was responsible for consolidating all 11 pages and maintaining their responsive design. HTML5 and CSS were used to build the site.

information architecture

After analyzing the provided data of 500 homes, we determined three factors to consider when retrofitting one's home: cost, environmental and lifestyle. This data allowed us to define our content, and determine the information architecture of our website through basic wireframing.

Wireframes at this stage were made by sketching or utilizing text to help us understand how the user would go through the website to achieve a given goal, whether it was learning about environmental factors or cost information.

high fidelity wireframes

After determining our desired user flow, we moved on to create higher fidelity wireframes of our website using Axure RP. In this phase we continued refining our content, created more detailed blocked elements and played with different interaction elements that we could incorporate.

reflections

Given time constraints provided by the course, ensuring that the website was perfectly responsive and consistent became difficult when combining all of the pages. By referring to the static mock-ups our team had made as a guide, we were able to finesse the code until we had a final product that provided a fairly consistent experience across the entire site.

Our team had initially planned to use a carousel format for going through the website. However, we found issues with usability and felt that it inhibited how we wanted our user to move throughout the site. After several iterations, we decided on a full-page styled website, so the user would be able to easily access information from the landing page, and would allow for more meaningful user flow.
