Sunday Morning Simulator

team

Individual project

tools

Java, Eclipse IDE, Adobe Illustrator, Violet UML Editor

my role

Developer, UX Designer, Storyboarder

brief

Design and code an interactive simulator using Java. A final course project for Multimedia Programming for Art and Design course (IAT 265). Fall 2016.

coding

This application was entirely coded in Java within Eclipse. The goal of the project was to implement object-oriented programming, such as inheritance, encapsulation and polymorphism. We explored using packages, factory and decorator patterns, as well as getter and setter methods when creating our own applications.

preview

A screen recording of Sunday Morning Simulator in window video, running through Eclipse. Recorded on my personal laptop.

storyboarding

The concept behind my simulator was to emulate a relaxing Sunday morning, where the user makes themselves a nice drink and a slice of toast before watching TV. I began by sketching general storyboards to establish the overall tone and cartoon-y design I wanted. These storyboards also detailed the types of user interaction I wished to implement. The various dragging, dropping, clicking, and arrow-key controls were determined in this stage.

uml diagram

By building UML diagrams, I was able to determine the application's architecture. This allowed me have a better understanding of the program's architecture by visualizing the relationships between the various portions of the code (functions, classes, packages, etc.) These diagrams were created using Violet UML Editor.

asset creation

All background and interactive elements in the application were made by myself, using Adobe Illustrator.

reflections

One challenge I faced when developing this application was realizing that my original project proposal was perhaps too ambitious, given the course's time frame and other academic responsibilities. I had to step back and honestly reflect on my progress and what I'd be able to achieve before the deadline. By honestly reflecting on the situation I was able to present a completed product while still maintaining the core functionality that I envisioned.

To take this project to the next level, my next step would be to debug and perfect the code, such as improving the hit detection on picky elements like the mug. I would also figure out what went wrong with importing in Processing's sound library, and implement music and sound effects to provide the extra bit of character that I had planned from the beginning of this project.
